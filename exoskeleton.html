Wearable Knee Exoskeleton Joint Control

Controlling a robot that is physically attached to a human is a completely different problem. This project tackled exactly that: designing and comparing controllers for a knee exoskeleton that has to track real gait motion while handling the unpredictable disturbances that come with human movement.

Why This Matters

Knee exoskeletons need to be smarter than they look

The core challenge: assistive joints face constant disturbances from human movement, weight shifts, and ground variations.

A wearable knee exoskeleton has one job: help the user move. But once someone is actually wearing the device, everything gets complicated. Human-robot interaction introduces unpredictable loads, motion variability, and modeling uncertainties that a simple PID controller

The question driving this project was straightforward: does adding Sliding Mode Control on top of PID actually improve robustness enough to matter? I wanted to answer that not just in theory, but with real gait data, real disturbances, and an honest look at the trade-offs.

The System

A simplified knee model grounded in real gait data

I did not invent the reference trajectory. I pulled it from the OpenSim Gait10dof18musc dataset, a publicly available biomechanical model used in gait research. I extracted a single right-knee gait cycle, smoothed it, and fed it into both controllers as the target to track. That grounding in real biomechanical data made the results more meaningful.

Fig. 1 — Knee angle trajectory extracted from OpenSim gait data. Blue = smoothed original, orange dashed = uniformly resampled reference used in simulation.

I modeled the joint as a single-DOF rotational system with inertia, damping, stiffness, actuator torque, and an external disturbance input. I made a deliberate choice to omit nonlinear gravity to keep the comparison tractable. It simplifies the model, but the results should be read with that in mind.

Honest assessment: Even the best controller in this project produced RMSE around 0.386 rad (~22°). Clinical benchmarks for acceptable wearable knee tracking are typically below 5° (0.087 rad). The hybrid controller is clearly better than PID in this model — but the gap to deployment-grade precision is significant. The simplified plant (no nonlinear gravity, no sensor noise) makes the problem easier than the real world. That's not a failure — it's a meaningful baseline for future work.
Fig. 4 — Robustness under ±20% parameter uncertainty. PID+SMC consistently outperforms PID, though the advantage narrows under mixed uncertainty.

The uncertainty testing added an important nuance. When inertia, damping, and stiffness were all varying simultaneously, the hybrid controller's advantage nearly disappeared. That finding was important to me because it stopped any overclaiming about SMC's universal robustness. It works well under some conditions and less so under others.

Takeaways

What this project actually taught me

Fig. 5 — Key conclusions across all test scenarios.

The biggest lesson for me was not about which controller won. It was about how to evaluate fairly. If I had only reported RMSE improvements without context about absolute error levels, model simplifications, and clinical benchmarks, the results would have been misleading. I was deliberate about designing the evaluation to tell an honest story.

Working with real gait data from OpenSim instead of a synthetic trajectory also added meaningful complexity. Extracting, smoothing, and resampling biomechanical motion data and then feeding it into a Simulink controller loop is a workflow that maps directly to how rehabilitation robotics research is actually done.

Where I would take this next: add nonlinear gravity, model sensor noise, extend to multi-DOF human-exoskeleton dynamics, and eventually move toward hardware-in-the-loop validation. The simplified model did its job as a comparison baseline. The scaffold is there to build something more realistic on top of it.
